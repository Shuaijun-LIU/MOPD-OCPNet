Framework Description

Detailed information about OceanEnv simulation environment and datasets

Overview

OceanEnv is a modular and scalable framework that unifies large-scale ocean data with advanced simulation tools for marine robotics training. The framework integrates high-resolution bathymetry, ocean physics, biodiversity, and microplastics data to create a comprehensive environment supporting marine robotics applications, pollution monitoring, and environmental research.

The dataset combines two primary sources:

  1. GEBCO 2025 global terrain data for bathymetry information
  2. Copernicus Marine Service (GOPAF) data for ocean currents and physical parameters

Through sophisticated interpolation techniques, these datasets have been harmonized to provide consistent spatial and temporal resolution across all variables.

Data Sources

GEBCO 2025 Terrain Data

Source: General Bathymetric Chart of the Oceans (GEBCO)

Description: The GEBCO_2025 Grid is a global terrain model for ocean and land, providing elevation data at 15 arc-second intervals.

Resolution: 15 arc-second grid (~450m spatial resolution)

Coverage: Global

Citation: GEBCO Compilation Group (2025). GEBCO 2025 Grid. doi:10.5285/1c44ce99-0a0d-5f4f-e063-7086abc0ea0f

Global Ocean Physics Analysis and Forecast (GOPAF)

Source: Copernicus Marine Service Information (CMEMS)

Description: Comprehensive ocean physics data including currents, temperature, and salinity.

Resolution: 0.083° horizontal resolution (~6.8km), interpolated to 15 arc-second (~450m) for final dataset

Coverage: Global ocean

Citation: E.U. Copernicus Marine Service Information (2025). Global Ocean Physics Analysis and Forecast (GOPAF). doi:https://doi.org/10.48670/moi-00016

Data Processing

Integration Methodology

The dataset was created through the following processing steps:

  1. Terrain Data Processing:
    • Extraction of GEBCO 2025 data for the region of interest
    • Filtering and quality control of bathymetry data
    • Conversion to standardized format
  2. Ocean Current Data Processing:
    • Acquisition of GOPAF data from Copernicus Marine Service
    • Merging of multiple GOPAF datasets
    • Interpolation of missing values using linear and nearest-neighbor methods
  3. Data Integration:
    • Harmonization of spatial grids using Radial Basis Function (RBF) interpolation
    • Temporal alignment of current data
    • Creation of unified NetCDF file with consistent dimensions

Data Format

File Format

The combined dataset is provided in NetCDF (Network Common Data Form) format, following CF-1.8 conventions. This format is widely used in climate, ocean, and atmospheric sciences and is compatible with most scientific data analysis tools.

Dimensions

Dimension Description Size Resolution
longitude Longitude coordinates Variable (depends on region) 15 arc-second (~450m)
latitude Latitude coordinates Variable (depends on region) 15 arc-second (~450m)
depth Depth levels Multiple levels 0.1m intervals
time Time dimension Multiple time steps 1 hour intervals

Coordinate Reference System

The dataset uses the WGS84 geographic coordinate system (EPSG:4326) with:

  • Longitude: -180° to 180°
  • Latitude: -90° to 90°

Data Quality

The dataset has undergone several quality control procedures:

  • Missing Value Handling: Missing values in the original data were filled using appropriate interpolation methods.
  • Consistency Checks: Variables were checked for physical consistency and realistic ranges.
  • Spatial Coherence: The integrated dataset was verified for spatial coherence across the domain.
  • Comparison with Observations: Where available, data was compared with in-situ observations for validation.

Known limitations:

  • Interpolation in areas with sparse data may introduce uncertainties
  • Coastal regions may have higher uncertainty due to complex bathymetry
  • Temporal resolution is limited by the source data frequency
